WAR INTEL Field Manual — Installation & Operation

A faction-wide enemy tracking tool covering player stats, fair-fight predictions and real-time flight monitoring. This guide walks you through installation on desktop and TornPDA, and how to read the dashboard once it's running.

§01 What is War Intel

Overview · What you'll see when it's running

War Intel is a small overlay that appears on every Torn page during ranked wars. It shows the enemy faction's members, their stats, and what they're currently doing — hospital timers, jail timers, and most importantly, flight activity.

During a war, knowing when an enemy is in transit is tactical gold. War Intel tracks every flight your enemies take — when they left, where they went, and when they're landing. It does this by combining observations from every member of our faction who has the tool installed, pooling that data on a shared backend so even if your browser is closed, the rest of the faction's observations are still being collected.

What you can expect

Once installed, you'll see a panel on the right side of your screen (desktop) or as a sliding card view (mobile) whenever there's an active ranked war. Between wars, it stays dormant. You'll see three views you can switch between:

  • By Stats — Enemies sorted by estimated battle stats, highest threats first.
  • By Timer — Enemies in hospital or jail, sorted by who's getting out first.
  • By Flight — Who's flying right now, who's abroad, who just landed.
Note — privacy War Intel is locked to Diamond Cartel members only. If someone outside the faction installs it, they'll see an "Access Denied" screen instead of any data. Your API key is hashed (never stored raw) and only used to verify your faction membership.

§02 Before you install

Required accounts & keys

You need one thing: a Torn API key. No signup, no account on any other service. Here's how to get one if you don't already have it:

  1. Open your Torn API preferences

    Go to torn.com/preferences.php#tab=api. You may be asked to log in.

  2. Create a new key

    Under "Create a key", pick Limited Access as the access level. Give it a comment like WarIntel so you recognise it later. Click "Create".

    Important Don't use Full Access — it exposes far more data than the tool needs. Limited Access is enough. If you already have a Limited key for something else, you can reuse it.
  3. Copy the 16-character key

    You'll paste this into War Intel's settings the first time you run it. Keep the preferences tab open for now.

§03 Installing on desktop

Chrome, Firefox, Edge, Safari · via Tampermonkey

On desktop, War Intel runs as a userscript. Userscripts are small JavaScript files that run inside specific websites — in our case, only on torn.com pages. To run them, you need a browser extension called Tampermonkey. It's free, very widely used, and safe.

  1. Install Tampermonkey

    Pick your browser:

    • Chrome / Brave / Arc
    • Firefox
    • Edge
    • Safari (macOS / iOS)

    Click "Add to Browser" or "Install". You'll see a new Tampermonkey icon near your address bar.

  2. Open the War Intel script link

    Open the installation link provided in the Diamond Cartel Discord (it'll be a .user.js URL). Tampermonkey will detect it automatically and open an installation prompt.

  3. Click "Install"

    You'll see a preview screen showing the script's name, author, version, and what permissions it needs. Click the big green Install button at the top.

    What permissions you're granting The script needs to talk to three servers: api.torn.com (for your faction data), ffscouter.com (for battle stat predictions), and the Diamond Cartel backend on Railway (for shared flight coverage). Nothing else.
  4. Refresh Torn

    Go to torn.com and refresh the page. The War Intel panel appears on the right side of the screen. If there's no active war, you'll see a note saying so — that's correct. Jump to §05 to enter your API key.

§04 Installing on TornPDA (mobile)

iOS and Android · via TornPDA's built-in script manager

If you play Torn on mobile through the TornPDA app, you don't need Tampermonkey — TornPDA has its own userscript manager built in.

  1. Install TornPDA if you haven't yet

    Available on the App Store and Play Store. Log in with your Torn credentials when prompted.

  2. Open TornPDA's settings

    Tap the More tab at the bottom (three dots or hamburger icon depending on version), then tap Userscripts.

  3. Add a new script

    Tap the + button (or "Add script" / "New" depending on version). You'll have two options:

    • Add from URL — paste the PDA install link from Discord
    • Add manually — paste the full script contents into the editor

    Either works. The URL method keeps you on the latest version automatically.

  4. Enable and save

    Make sure the toggle is ON and tap Save. The script activates the next time you load any Torn page inside the app.

  5. Reload Torn inside TornPDA

    Go back to the main Torn view and pull down to refresh. The War Intel panel appears. Enter your API key in settings — see §05.

Separate script, same backend The desktop and TornPDA versions are two different .user.js files, optimised for their respective environments. They share the same Diamond Cartel backend, so flight history you observe on desktop is visible on mobile and vice-versa.

§05 First run & settings

Entering your API key · One-time setup

The first time War Intel runs, it'll prompt you to enter your API key because it needs to verify you're a Diamond Cartel member before showing any data.

  1. Open the Settings panel

    Click ⚙ Settings in the War Intel title bar. A panel expands with two input fields.

  2. Paste your Torn API key

    Into the Torn API Key field. It should be exactly 16 characters. If you pasted extra whitespace or a partial key, you'll see an alert.

  3. FFScouter key — optional

    The FFScouter Key field is optional now that the shared backend handles this for you. You can leave it blank. If you already have one and paste it here, tick the data-policy checkbox to confirm you've read FFScouter's data policy.

  4. Click "Save & Fetch"

    War Intel stores your key locally (never sent anywhere except for auth), contacts the backend, verifies faction membership, and loads the enemy faction if there's an active war.

You're done Your settings are remembered. You won't need to touch this panel again. The green dot in the title bar is your confirmation that the backend handshake succeeded — more on what the dot means in the next section.

§06 Reading the dashboard

The three views · The status dot · Flight history

The status dot

Next to the "WAR INTEL" logo in the title bar, there's a small coloured dot. It tells you at a glance what mode the tool is operating in:

  • Green — Connected to the shared Diamond Cartel backend. You're seeing faction-wide pooled observations. This is the normal state.
  • Orange — Backend is unreachable, the tool is falling back to direct Torn API polling. Still works, just without shared coverage. Usually self-heals within a minute or two. Hover/tap the dot to see the error reason.
  • Red — Both paths have failed. Something is genuinely wrong — check your API key, check your internet, or report it in Discord.
  • Grey — Initial connection, usually only for a few seconds on first load.

Switching between views

Three tabs sit under the war header: ⚔ By Stats, ⏱ By Timer, ✈ By Flight. Tap or click any of them to switch.

⚔ By Stats

Every enemy sorted by FFScouter's battle-stat estimate (highest first by default). Click a column header to re-sort. The FF column shows the fair-fight multiplier — green is easy, yellow is even, red is dangerous. The Dist. column shows the distribution confidence from FFScouter.

⏱ By Timer

Enemies grouped by status. "In Hospital / Jail" sorted by who's getting out soonest. "Out Soon" are ones whose timer just expired but the backend hasn't re-polled to confirm their status. "Available Now" are okay and attackable.

✈ By Flight

Three sections: Currently Flying (in transit, with live landing countdowns), Abroad (currently in another country), Recently Landed (landed in the last 30 minutes). The route column shows origin and destination — for example TRN → JAP for an outbound flight to Japan, or SWI → TRN for a return flight.

Clicking a player

In any view, you can click (desktop) or tap (mobile) a player's row to expand their flight history panel. You'll see every flight they've taken that the backend has observed, with times and destinations. This is where pattern-spotting happens: "this guy always flies Japan around downtime," or "they always go Cayman for stat training."

Building up history Flight history builds up over time. When the tool was first deployed, everyone's history was empty. After a week of watching a war, players will have 5-10 entries each. After a month of on-and-off coverage, you'll have genuinely useful behavioural patterns.

The Fly / Refresh / Export buttons

✈ Fly opens the Torn travel agency in a new tab — handy if you see an enemy land and want to chase them. ↻ Refresh forces an immediate refresh instead of waiting for the next automatic poll. ⬇ Export dumps the current roster to a .txt file for sharing in Discord or wherever.

§07 Things to check if it's not working

Most common issues & their fixes

If something's off, scan this list first before asking in Discord — nine times out of ten the answer is here.

The panel doesn't appear at all
Tampermonkey isn't running, the script isn't enabled, or you're on the wrong page.
Click the Tampermonkey icon in your browser toolbar → make sure the switch at the top is green (enabled). Click "Dashboard" and confirm Torn War Intelligence Dashboard is in the list and toggled on. Then reload a torn.com page.
"Access Denied — Diamond Cartel members only"
Your API key belongs to someone whose faction ID isn't 48822.
Either you're not in Diamond Cartel (in which case the tool isn't for you) or you pasted the wrong API key — possibly a friend's or an old one from before you joined. Double-check the key in settings is from your current account.
The status dot stays grey
No API key entered yet, or it's still on the very first connection attempt.
Open Settings, check your key is there. If it is and the dot stays grey more than 10 seconds, click Refresh. If still grey, something is blocking the connection — check the "Orange dot" row below.
The status dot is orange
Backend is unreachable. Tool is still working, just using direct Torn polling.
Usually self-heals within a minute. Hover (desktop) or tap (mobile) the dot to see the error. "timeout" or "network" means the backend is briefly down — wait a bit. If it's been orange for more than 10 minutes, mention it in Discord — it's probably a Railway issue on our side.
"No active ranked war detected"
Correct behaviour when we're not in a war.
War Intel only activates during ranked wars. Between wars the panel shows this message and does nothing. It'll wake up automatically within 2 minutes of a new war starting.
Flight history is empty for every player
Either the war just started, or you're on orange (fallback) mode.
Flight history is accumulated by the backend over time. A fresh war with a new enemy means everyone starts with 0-1 flights logged. On orange/fallback mode, history is local to your browser only — it'll merge with the backend's full history once you're back on green.
FFScouter stats column shows "—" for everyone
Backend's FFScouter cache hasn't populated yet, or the war just started.
The backend refreshes FFScouter data every 5 minutes. Give it a single 5-minute cycle after war detection and the column will fill in. If it stays empty for 15+ minutes, report in Discord.
Everything worked, then stopped after a Torn update
Torn occasionally changes how the API responds, which can briefly break the fallback path.
The backend path is resilient — if your dot is green, you're fine. If it's orange and nothing works, check Discord for an announcement of an updated script version.

Still stuck?

Post in #war-intel on Discord with: the colour of your dot, the error message if any, and whether you're on desktop or TornPDA. Screenshots help a lot. Don't post your API key — ever.
